## Your Mission

Create an organized README.md index for all generated architect deliverables.

## Prerequisites

1. Look for generated deliverables in `docs/software-architect/` (or the custom output_dir from fa-context.json):
   - `fa-context.json` — required
   - `deliverables/proposal/proposal.md` — optional
   - `prototype/` — optional
2. If no deliverables found: "No deliverables found. Run `/software-architect:deliver` or individual skills first." Then stop.
3. Read `fa-context.json` for `output_config`.

## Action

Create/update `docs/software-architect/README.md` as an index organized by audience:

```markdown
# {{project.name}} — Architect Deliverables

Generated on {{metadata.generated_at}}

## By Audience

### Client / Direction
| Deliverable | MD | DOCX | PDF |
|------------|-----|------|-----|
| Technical Proposal | [proposal.md](deliverables/proposal/proposal.md) | [proposal.docx](deliverables/proposal/proposal.docx) | [proposal.pdf](deliverables/proposal/proposal.pdf) |

### Everyone
| Deliverable | Location |
|------------|----------|
| HTML Prototype | [prototype/index.html](prototype/index.html) |
| Architecture Diagram | [diagrams/](diagrams/) |

## Quick Start

1. Read the [Technical Proposal](deliverables/proposal/proposal.md) for project overview
2. Open [Prototype](prototype/index.html) in your browser

---
*Generated by the [architect](https://github.com/<user>/architect) Claude Code plugin*
```

Mark each file with ✅ if it exists or ❌ if not yet generated.

## Output

1. Write `docs/software-architect/README.md`
2. Present a summary:
   > "Export complete. Deliverables index at `docs/software-architect/README.md`:
   > - [list of what was generated with ✅]
   > - [list of what's missing with ❌ and hint to generate it]
   > Total: [X] deliverables + [Y] prototype pages"
